Three Ways to Load Data

Department of Mathematics, Applied Mathematics, and Statistics
Case Western Reserve University

Overview

In STAT 312/312R we will routinely import data for descriptive statistics, inference, and regression (as in Engineering Statistics by Montgomery, Runger, and Hubele). Below are three reliable workflows for getting data into R. For reproducibility, prefer a script-based approach (Methods A or C) inside an RStudio Project.

A) Local CSV using readr::read_csv()

Download the sample dataset: thickness.csv (stored at /projects/datasets/thickness.csv). If your script lives in /projects/R/, use the relative path ../datasets/thickness.csv. 

library(readr); library(dplyr)

# Inspect your current working directory if needed
getwd()

# Read from a relative path (script in projects/R/)
thick <- read_csv("../datasets/thickness.csv")

# Quick checks
glimpse(thick)
summary(thick)
thick %>% summarize(n = n(), mean = mean(value), sd = sd(value))

If you see “cannot open the connection” or “No such file or directory”, verify the path and filename, including capitalization and file extension.

B) Directly from a URL

When a dataset is hosted online (e.g., a raw CSV on GitHub or a departmental server), you can read it directly via HTTPS. This is convenient for sharable examples and keeps your script portable. 

library(readr)

url_csv <- "https://example.org/path/to/thickness.csv"   # replace with a real raw CSV URL
thick_url <- read_csv(url_csv)

# Confirm shape/content
dim(thick_url); head(thick_url)

For GitHub files, use the “raw” URL. Note that network hiccups can cause intermittent failures; having a local copy is helpful for class.

C) Excel files using readxl::read_excel()

Many engineering datasets come in Excel format. The readxl package imports .xlsx without requiring Excel to be installed. Place the spreadsheet in a data folder within your project. 

library(readxl); library(dplyr)

# Example: read a specific sheet and range (adjust to your file)
# File placed at /projects/datasets/thickness.xlsx
dat_xlsx <- read_excel("../datasets/thickness.xlsx", sheet = 1)

# Optional: rename columns and compute summaries
dat_xlsx <- dplyr::rename(dat_xlsx, value = 1)
dat_xlsx %>% summarize(n = n(), mean = mean(value), sd = sd(value))

If your decimal separator is comma (e.g., 2,01), clean or set locale accordingly before import.

Verify the Import and Plot

library(ggplot2)

# Using the CSV loaded in Method A
ggplot(thick, aes(x = value)) +
  geom_histogram(bins = 12) +
  labs(x = "Thickness (mm)", y = "Count",
       title = "Process Thickness Distribution")

This mirrors the early chapters in the text: summarize center/spread and visualize variability.

Common Issues & Tips

  • File not found: Check the relative path from your script location. Prefer RStudio Projects.
  • Wrong delimiter: Use read_delim() with the correct delim (e.g., ;).
  • Headers/encoding: Set col_names = TRUE/FALSE or locale = locale(encoding = "...").
  • Missing values: Pass na = c("", "NA", "-999") if needed.
